## post-page — shipped 2026-05-20

**Merge commit:** `dd98285`
**Plan-removal commit:** `7233feb`
**Feature branch (deleted):** `feature/post-page`

### What shipped
- Backend: `PublicPostSource` + `PublicPostDetail` Pydantic schemas in `backend/schemas/public.py`
- Backend: `GET /public/posts/{slug}` endpoint in `backend/routers/public.py` — only returns `published` posts, eager-loads sources via `selectinload`
- Backend: 4 new tests in `tests/test_public_posts.py` covering detail shape, 404 on missing slug, 404 on non-published slug, no admin field leakage
- Frontend: `PublicPostDetail` + `PublicPostSource` TypeScript interfaces + `getPublicPost(slug)` + `NotFoundError` in `lib/public-api.ts`
- Frontend: `components/public/post-body.tsx` — react-markdown + remark-gfm with custom Chakra Petch headings and Inter body
- Frontend: `components/public/share-bar.tsx` — X, LinkedIn, Copy Link buttons (dual-chamfer outline); Copy shows `Copied ✓` for 2 seconds
- Frontend: `components/public/sources-list.tsx` — collapsible sources section (toggle open/closed)
- Frontend: `app/(public)/blog/[slug]/page.tsx` — server component with `generateMetadata` for OG tags (title, description, og:title, og:description, og:type, og:url, og:publishedTime, twitter card)

### Manual tests for the user to run
- Boot the stack (`docker compose up -d --build && cd frontend && npm run dev`) and navigate to `/blog/[any-published-slug]`
- Confirm: post title (Chakra Petch, large), publish date + read time + tag pills in header, rendered markdown body
- Click X button — confirm opens `https://x.com/intent/tweet?url=...&text=...`
- Click LinkedIn button — confirm opens `https://www.linkedin.com/sharing/share-offsite/?url=...`
- Click Copy Link — confirm `Copied ✓` appears then reverts to `Copy Link` after ~2 seconds
- Expand Sources — confirm accordion opens with linked titles + publisher + date
- View page source — confirm `<meta property="og:title">`, `<meta property="og:description">`, `<meta property="og:url">`, `<meta property="og:type" content="article">` present

### Pending human checks

- Share button hover states (accent color dims correctly)
- Sources collapsible open/close animation

### Architectural changes
- New `GET /public/posts/{slug}` public endpoint — consistent with the existing `/public/posts` list endpoint pattern. No new tables, no new external dependencies.
- `PublicPostDetail` exposes `content` and `meta_description` to the public surface — deliberate (needed for markdown rendering and OG tags). Admin-only fields (`status`, `publishing_mode`, `generation_attempt`, `scheduled_at`) are excluded.
